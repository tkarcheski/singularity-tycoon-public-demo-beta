# Universal Basic Compute + Universal Basic Income (design spec)

Tyler's ask 2026-07-02: "add allocation for universal basic compute,
universal basic income where we can steer profits into making more jobs and
people happy." Stacked on the cooling/synergy branch (PR #39).

## Problem

Public sentiment is currently something that happens *to* you (jobs displaced
by compute) with only indirect levers (retraining centers, worker pods). The
player can't deliberately invest in the public. Meanwhile sentiment already
pays: token demand scales with happiness (`DEMAND_PER_SENTIMENT`), goodwill
cuts upkeep 15%, protest halves compute. Giving back should be a real
strategy, not just damage control.

## 1. Universal Basic Compute — 4th allocation slider

"Public compute" joins Sell / Research / Self-improve in the normalized
allocation group. The donated share of TFLOPS:

- earns no revenue and no RP (normalization handles this automatically),
- **doesn't displace jobs** — displacement now counts only non-donated
  compute: `computeAdj × (1 − alloc.ubc) × JOBS_DISPLACED_PER_TFLOPS`,
- adds a direct sentiment-target bonus: `min(30, donatedTflops × 0.5)` —
  free AI for schools, clinics, and civic services.

## 2. Universal Basic Income — profit dividend slider

A separate slider (0–30%, not part of the compute normalization — it's
cash, not tokens): `state.ubiShare` of gross sell revenue is paid out as a
public dividend each tick:

- deducted from revenue before it reaches cash,
- funds jobs: `jobsFunded = ubiSpend × 0.6` added to the jobs ledger, which
  feeds the existing sentiment target (`50 + netJobs`) and the HUD jobs stat.
  (Jobs use the previous tick's spend — steady-state correct, 1-tick lag.)

## The loop this creates

Give away compute/profits → sentiment climbs → token demand raises price
(up to ×1.4 at 100) + goodwill upkeep rebate → selling the *remaining*
share earns more per TFLOPS. At high compute, generosity approximately pays
for itself — which is the point (and mirrors the entropy tension: the AI
that shares wins the public; the AI that hoards fights protests).

## Constants (tunable)

| | |
|---|---|
| `UBC_SENT_PER_TFLOPS` | 0.5 sentiment pts per donated TFLOPS |
| `UBC_SENT_CAP` | 30 pts |
| `UBI_JOBS_PER_DOLLAR` | 0.6 jobs per $/s of dividend |
| `UBI_MAX_SHARE` | 0.30 |

## UI

- Allocation panel: 4th row "Public compute"; below the group, a "UBI —
  profit share" row (0–30%) with its own % label.
- The alloc-note line reports live effects: `UBI $X/s → +Y jobs · Public
  compute Z TF → +S mood`.
- `ubiShare` joins `SAVE_KEYS`; old saves default it to 0 and `alloc.ubc`
  guards with `|| 0`.

## Verification

pytest: 4 alloc sliders + UBI slider render; raising UBC lifts sentiment
above the no-UBC baseline and cuts revenue share; UBI spend raises netJobs
and reduces cash growth; save roundtrips `ubiShare`; zero console errors.
